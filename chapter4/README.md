Amazon EC2를 생성하는 CloudFormation 코드를 담고있습니다.

VPC.yml 파일의 구성을 시각적으로 도식화하여 설명해 드릴게요.

**VPC 네트워크 구조도 설명**

제공해주신 VPC.yml 파일을 기반으로 구성한 네트워크 구조도는 다음과 같습니다.

* **VPC (CidrBlock: 10.0.0.0/24)**: 전체 네트워크를 담고 있는 가장 큰 테두리입니다. 모든 리소스는 이 안에 위치합니다.  
* **Availability Zone (1a, 1b)**: 두 개의 가용 영역으로 나누어 이중화 구성을 시각적으로 보여줍니다. 각 서브넷은 특정 AZ에 속해 있습니다.  
* **Subnet Tiers**:  
  * **Public Subnets (Green)**: 인터넷과 직접 통신하는 영역입니다. 인터넷 게이트웨이(IGW)로 향하는 라우팅(PublicRTB)이 설정되어 있으며, NAT 게이트웨이(NAT GW)도 이 영역에 위치합니다.  
  * **Web Subnets (Orange)**: 실제 웹 서버가 위치하는 프라이빗 영역입니다. 아웃바운드 인터넷 통신을 위해 NAT 게이트웨이로 향하는 라우팅(WebRTB)이 설정되어 있습니다. (인터넷에서 직접 접근 불가)  
  * **Datastore Subnets (Purple)**: 데이터베이스와 같은 격리된 영역입니다. 외부 인터넷 통신 없이 로컬 통신만 가능한 라우팅(DatastoreRTB)이 설정되어 있습니다. (가장 보안이 강력한 영역)  
* **Internet Connectivity**:  
  * **IGW**: Public 서브넷의 아웃바운드 인터넷 통신을 담당합니다.  
  * **NAT GW**: Web 서브넷의 아웃바운드 인터넷 통신을 위해 Public 서브넷에 배치됩니다.

이 도식화된 이미지를 통해 VPC.yml 파일이 생성하는 인프라의 논리적이고 물리적인 구조를 한눈에 파악하실 수 있습니다.
